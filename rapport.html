Réalisation du wiki

Après avoir lu le code source de PtiWiki, nous avons commencé l'implémentation du wiki par la mise en place du système de gestion des comptes. Comme l'exige l'énonce, les comptes sont divisés en comptes d'administrateur et comptes d'utilisateur. Les comptes d'administrateur peuvent effectuer des opérations de suppression sur les comptes d'utilisateur, tandis que les comptes d'utilisateur seront directement liés à l'interface wiki. En fonction des fonctions exercées, nous avons divisé les dossiers en plusieurs :

login.php - l'écran de connexion principal, principalement utilisé pour collecter des informations
adminPage.php - l'écran d'administration, utilisé pour afficher les informations relatives au compte de l'utilisateur et pour effectuer des actions de suppression.
createAccount.php - l'écran de création de compte, qui crée de nouveaux comptes d'utilisateur
server.php - le fichier de traitement dorsal du serveur, qui est responsable de l'exécution d'opérations et de fonctions spécifiques au wiki.
database.php - la connexion à la base de données du wiki

Dans l'implémentation, nous avons ajouté plusieurs écrans pour afficher les messages d'erreur
Ensuite, nous avons créé un fichier text.sql pour importer le fichier texte dans notre base de données.

Ce que nous avons appris pendant réaliser ce TP

Une implémentation concrète d'une connexion php à une base de données sql.

Sauts multiples de l'interface php, implémentation de l'affichage des messages d'erreur.

Méthode d'importation de fichiers texte dans une base de données sql.
